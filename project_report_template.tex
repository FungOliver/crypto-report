\section{Basic Description of Ring LWE}
LAC is, as many post quantum, algorithms based on Ring Learning With Errors [Citation Needed]
Ring Learning with errors itself is an evolution from Learning with Errors [Cite REGEV].

Learning with Errors is based on the following simple principle for Key generation:
We work in integers modulo q. We have a Matrix A and a secret vector s plus a small random vector e
We calculate b = As + e . The public key then is (A,b) and the private key is (A,s) the random error e is discarded but because of it the private key cannot be easily reconstructed from the public key. The Matrix A can be seen as a Basis of a Lattice and then it becomes clear that finding s from b is the Closest Vector problem.

Furthermore it was possible to show that for certain kinds of Lattices the existence of a Algorithm that solves b=As +e in the average case would imply an algorithm that Solves the Shortest Vector problem in the worst case, which would imply P=NP.

Therefore LWE Encryption Schemes are considered quite secure. The main problem with them is that the matrix A has to be quite large to provide sufficient security, this creates performance problems and is why Ring encryption was developed. \section{Description of Parameter Choices}
Most asymmetric lattice-based encryption schemas use a 
similar structure and are based on Ring-LWE (RLWE) and Learning with Errors (LWE). 
The security of these schemes relies on the hardness of 
the mathematical problems, such as the Shortest Vector Problem (SVP)
 and the Closest Vector Problem (CVP), 
 which are believed to be difficult to solve even with quantum computers.
 However, those problems do not give hints on how to choose the 
 parameters of the encryption scheme,
 Choosing the right parameters may be deciding factor between
 proposals chosen for the NIST post-quantum cryptography standardization process.
 In this section, we will discuss the chosen parameters for LAC and the rationale behind them.

Choosing the right modulus is important as it determines 
the size of the payload. A smaller modulus leads to smaller keys and ciphertexts, 
which is desirable for efficiency. 
This comes at the cost of increased probability of decryption errors, 
as the noise in the ciphertext may exceed the modulus during decryption.
However, a larger modulus has the advantage of allowing for more noise tolerance, which 
decreases the probability of decryption errors.

Unlike earlier schemes that used relatively large modulo such as q=12289.
LAC tries to use byte level modulus, which allows for more efficient implementations.
The authors argue that using modulos smaller than byte level 
does not make sense from a performance perspective, as bytes are the smallest
addressable units in most computer architectures.

The papers considers three types of byte-level modulus: 
power-of-two modulus (q=256), max-split modulus (q=257), and min-split modulus (q=251).
For this to work we can observe that our noise is small enough to be tolerated by the byte level modulus.

\begin{itemize}
  \item Power-of-two modulus (q=256): This choice allows for efficient implementation, as the modulus operation can be performed by simple bitwise operation.
  \item Max-split modulus (q=257): This choice allows for maximum factorization of the polynomial xn + 1
  \item Min-split modulus (q=251): This choice allows for minimum factorization of the polynomial xn + 1
\end{itemize}

The authors chose the min-split modulus (q=251) for the main version of their scheme submitted to NIST-PQC, as they argue that less algebraic structure reduces the attack surface.

\section{Attacks}
There has been a number of attacks on LAC.
We will now look at three attacks.

CCA Attack \cite{10.1007/978-3-030-34578-5_4}: LAC allows for a small 
chance of decryption failure, 
which is a common feature in 
lattice-based encryption schemes.
If the noise is too large, decryption may fail,
and the decryption algorithm will output 
an error instead of the plaintext.
Attackers can exploit this decryption failure
to gain information about the secret key.
By crafting specific ciphertexts that
allow them to observe whether decryption fails or not,
attackers can infer information about the secret key.


D'Anvers et al. showed that LAC is vulnerable to a timing side-channel 
attack stemming from its BCH error-correction 
decoding step. Because the decoder runs
in variable time depending on whether errors 
are present, an attacker can distinguish 
ciphertexts that induce decoding errors
from those that do not. 
Using carefully crafted chosen
ciphertexts and timing measurements,
the attacker can infer coefficients
of the secret key and eventually recover 
the full secret key,
thereby breaking the intended
IND-CCA security of the scheme.
The authors demonstrated full key
recovery for all LAC parameter sets 
in under two minutes using fewer than 
$2^10$ decryption queries.

A third attack is the misuse of protocol.
